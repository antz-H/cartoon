# 叙事 memory 协议

## 核心原则

`memory.md` 唯一用途：**提升下一轮定主题 / 写即梦 prompt 的质量**。跨轮原则蒸馏，非案情摘要或数据台账。

**覆写哲学：原则先于案情**——只写「删掉作品序号后仍对下一张有用」的短句。

## 结构化格式

用标签行 + `|` 分隔独立规则，每条规则可独立增删，合并时逐条匹配而非整句覆写：

```
A:<类型化未决，≤1句>
B:<画面规则1>|<规则2>|...
C:<prompt规则1>|<规则2>|...
T:<标题规则1>|<规则2>
```

- **A** 弧位/未决：≤1 句类型化，禁台账
- **B** 画面与构图：维度级规则（机位/留白/轮廓/钉位/禁项等）
- **C** Prompt 写法：指令形态级（五段习惯/否定项/钉尾/词密度）
- **T** 标题与人感：句式策略、小图自检要点

**禁写**：整段 prompt、长对白、剧情全文、单次运营数据原文、作品序号/路径作独立信息。角色否定（芽仔≠写实男童等）已在 prompt 角色固定段内置，**不在 C 槽重复**。

**硬约束**：整文件 **≤300 字**（含标签、分隔符、标点空格），超出则删并合并同类项；优先删单次数据、过期弧位、被推翻偏好。

## 读

每次开跑先读（若存在）：
- `memory.md`（仓库根 `cartoon/memory.md`，或用户指定路径）
- 近 2～3 篇 `作品/*.md` 的本页情节与留白

`memory.md` 不存在时不创建，仅确认后新建。前一日数据（曝光/点击率等）作可选语境对照，**弱信号原则**微调维度，不推翻主目标；未经蒸馏+确认不写进 memory。

## 询问（必须）

交付后依次询问：

> 请给本次 **即梦正向 prompt 有效性** 打分：**0～10**（0=几乎无效或严重跑偏，10=高度贴合、你只需微调即可出片）。分数会用来**调整** `memory.md` 里关于「怎么写 prompt 才有效」那几句总结。

> 是否把本次反馈**并入** `memory.md` 的**滚动总结**（整文件总字数 **≤300 字**；保存时为覆盖写回，但会与旧版**合并蒸馏**为**原则级、可迁移**短句；**不写**篇号数据路径、不单话复述剧情）？回复 **是 / 否**。

用户可合并回复如 `8 分，是`。**不得**在用户明确「是」之前写入。

## 写 · 用户确认「是」之后

**自检**：用户已确认？→ 已读旧版？→ 预估 ≤300 字？→ 正文非剧情/数据为主体？（任否→停或先修再写）

**结构化合并步骤**（替代旧版散文分类）：

1. **读取**当前 memory.md → 按 A:/B:/C:/T: 标签解析，`|` 分割为独立规则列表
2. **逐条匹配**：新增量中的每条规则 → 找旧列表中同主题条：
   - 同义 → 保留更强的那条（评分高者优先）
   - 矛盾 → 以本轮为准替换旧条
   - 新主题 → 追加到末尾
   - 旧条无对应新增量且仍成立 → **默认保留**
3. **具象扫描**：删作品序号、路径、纯数字串、仅本篇成立的道具句；或升格为维度级规则
4. **压缩**：若超 300 字，按优先级删：A 旧钩（已落地）→ T 弱句 → B 单次规则 → C 弱句；**不**优先删 C 核心习惯与 B 反复验证的构图规则
5. **写回**：按 A/B/C/T 顺序重组为标签行格式，`|` 连接，整文件一次性覆盖
6. 回复说明已重写、路径、约 X 字/300

### 依评分调整 C 区

- **9～10**：最管用一条压成短语写进 C，删旧弱句
- **7～8**：轻量强化半句
- **4～6**：1 条待改进
- **0～3**：1 短句避坑

### 首次建立

首行：`# 墨团与芽仔 · memory`，其下 C/B 为主、A 一句类型化，整文件 ≤300 字。
