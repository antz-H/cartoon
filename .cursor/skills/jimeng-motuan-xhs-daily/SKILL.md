# JiMeng · 墨团与芽仔 · 每日小红书稿

## 先读（按需展开）

1. **[references/memory-protocol.md](references/memory-protocol.md)**：`memory.md` 为**跨轮原则与偏好**（**覆写哲学：原则先于案情**；**整文件≤300字**、覆盖写回 + **槽位合并**）；**非**剧情/数据/路径台账；含 **0～10** 与写入询问（**未确认禁止写入**）。
2. **[references/iteration-rubric.md](references/iteration-rubric.md)**：**落盘前反思闭环**（prompt 五段/留白/可画性/近页防同构、`作品/` 三节与序号、`memory` 写入闸门）；不过线时允许**静默修订一轮**再交付。
3. **[references/xhs-and-tone.md](references/xhs-and-tone.md)**：标题字数、正文气质、可选笔记附录、缩略图极简自检。
4. **[references/Prompt-template.md](references/Prompt-template.md)**：五段拼装与留白版式原文；**留白段结构勿改**，`[本页情节]` **以 `竖版内页。` 起笔**写当页正文，**勿**重复粘贴画风段。

钉尾、模型常漏留白等：**按需**查阅仓库根目录 `即梦绘本Prompt总结.md`（尤其 §4.4、§2.4），本 skill 不重复全文。

## 默认输入（仓库根 `cartoon/`）

- 累积指导：`memory.md`（仓库根，与 `作品/` 同级；**总字数≤300**，见 protocol；读时当作**下一轮定主题 / 写即梦 prompt 的原则与偏好**，写时当作**原则级合并覆写**，不是案情台账）
- 近页：`作品/*.md`（取最近 2～3 篇扫 `本页情节` 与留白，防同构）
- **前一日数据反馈（可选）**：用户消息里粘贴的曝光/点击率/互动等；与 memory、近页**对照**用，**弱信号**微调标题与首图相关维度；写入 memory 时只保留**蒸馏原则**（详见 [references/memory-protocol.md](references/memory-protocol.md)「前一日数据反馈」）；**未经确认写入不落地到 memory.md**
- 若用户另有 memory 路径：以用户消息为准，并在 SKILL 执行中替换默认路径

## Workflow（顺序固定）

### 1. 定主题（连贯 + 角色 + memory）

- 用 `memory.md`、近页 `作品/`（及用户本轮提供的弧/大纲，若有）对齐：**上一拍未决**、情绪走向、重复母题。
- 本拍须 **单场景、可画、小具体**；与墨团（沉、稳、墨）与芽仔（小、灵、笋）**行为动机一致**。
- 过 [references/xhs-and-tone.md](references/xhs-and-tone.md) 的「信息流极简」：相对近页至少变一项（天色/机位/道具/身体关系）。
- **同步写清「为什么是本主题」**：在脑中成稿一段**选型理由**（承接、防同构、memory 有无、可画性、角色动机），**必须**在步骤 3 落盘为 `## 主题与选型理由`（见下），便于你事后对照「发给生图模型的 prompt」复盘。

### 2. 定小红书主标题

- **3～10 个汉字**，概括本拍；白话、自解释；规则见 [references/xhs-and-tone.md](references/xhs-and-tone.md)。
- **不**进入即梦正向 prompt。

### 3. 生成即梦 AI 正向 Prompt（**必须落盘 .md**）

- **落盘前（强制）**：按 [references/iteration-rubric.md](references/iteration-rubric.md) 表格 **A（prompt）+ B（md）** 自检；任一项不过线则 **只改 prompt 或 md 内文字** 并 **再跑一轮自检**（最多两轮静默修订）。仍卡壳（如弧与近页硬冲突）→ 聊天用 1～2 句说明卡点；**禁止**谎称已写入 `作品/`。
- 五段顺序与 [references/Prompt-template.md](references/Prompt-template.md) 一致：`[角色固定段]` → `[画风段]` → `[留白指令段]`（选一）→ `[本页情节]`（以 `竖版内页。` 起笔写当页）→ `[质量词：高清、宣纸质感]`。
- 需要钉尾时按 `即梦绘本Prompt总结.md` §4.4 在 **发给即梦的整段末尾** 单独追加一句（有底条钉底、仅顶条钉顶，勿混用）。
- 主画面 **不要求** 模型内生成汉字；配文走留白后期。
- **落盘（强制）**：在拼装完成后，**新建** `作品/{两位序号} {白话标题}.md`。**序号**取 `作品/` 现有文件中最大两位序号 **+1**；若尚无作品则从 `01` 起。**勿覆盖**已有序号文件。`{白话标题}` 与步骤 2 主标题语义一致即可（文件名可读优先，不限死字数）。
- **文件内容至少**包含下列章节（**顺序固定**，便于你打开一篇从上到下读：先理由，再标题，再即梦）：
  1. `## 主题与选型理由`：**人话**写清「为什么定这一拍」（**不写进**即梦 prompt）。**至少**覆盖五条短意（可用小标题或短段，禁流水账堆形容词）：①承接上拍 / 未决钩子；②相对近 2～3 页 `作品/` 改了哪一项（天色 / 机位 / 道具 / 身体关系，点名序号或篇名）；③与滚动 `memory.md` 的对齐或张力（若尚无该文件，写一句 **「memory 尚未建立」**）；④本拍为何**单场景、可画**；⑤墨团 / 芽仔**动机各一句**。此处是「选题说明」，与下一节「发给模型的指令」分离。
  2. `## 小红书主标题`：一行，步骤 2 的 3～10 字标题。
  3. `## 即梦正向 Prompt`：节首可加一行说明，例如：`以下为发给即梦（生图大模型）的完整正向 prompt，与上节选题理由对应。` 其下为**完整**正向 prompt 正文（推荐用单个 markdown 代码围栏包裹，便于整段复制；围栏内勿再嵌套同语言三反引号）。若本 run 还产出了钉尾，钉尾包含在该节全文内。**本节即「写给大模型的 prompt」存档**，与 `## 主题与选型理由` 成对保留在同一 md。
- **旧稿**：此前已落盘、无 `## 主题与选型理由` 的文件不必自动回写；**自本 skill 生效后起** 新建 `作品/*.md` 一律含该节。
- **可选**追加至同一 md（若本 run 已写）：留白两行对白、`小红书 · 对白释义`、发布包装要点等——遵守 [references/xhs-and-tone.md](references/xhs-and-tone.md)。
- 在聊天中给出 **可复制的完整正向 prompt**（或说明「与 md 中一致」），标注「以下为即梦粘贴区」，并**写明已写入**的 `作品/...md` 路径；选题理由以 md 中 `## 主题与选型理由` 为准（聊天里可 2～4 行缩略复述）。

### 4. 总结并 **询问**（评分 + memory）

- 用短列表汇总：**主题一句、主标题、本拍要点、未决钩子**。
- **必须**使用 [references/memory-protocol.md](references/memory-protocol.md)「询问（必须）」中的两段原文（先 **0～10 即梦 prompt 有效性**，再 **是否并入 `memory.md` 滚动总结 ≤300 字**）；**停止**，等用户回复。
- 说明口径：**分数衡量发给即梦的整段正向 prompt** 对控图/出图的帮助，不是给小红书配文打分。

### 5. 用户确认「是」之后

- 写入前再过 [references/iteration-rubric.md](references/iteration-rubric.md) **节 C**（闸门：已确认、≤300 字、槽位合并、禁剧情顶替）。
- 按 [references/memory-protocol.md](references/memory-protocol.md) **读取旧版（若有）→ 槽位合并（弧位 / 选题与画面偏好 / Prompt 写法）→ 具象扫描 → 整文件重写** `memory.md`（或用户指定路径）；覆写目标见 protocol **「覆写哲学：原则先于案情」**：**C/B 优先**、**A 一句类型化**、**禁篇号数据路径与单话案情占主篇幅**。**全文总字数 ≤300**（含标题标点空格），超出则删并合并同类项；**默认继承**旧版中仍成立的**原则句**，除非本轮反馈否定或与近页 `作品/` 矛盾。
- 把用户 **0～10 分** **转写**进 **C 区抽象指令**（高分：一条可脱离本篇仍成立的强化；低分：一条避坑），**禁止**用剧情、数据、路径占满篇幅；**禁止**贴整段即梦 prompt（全文以当页 `作品/*.md` 为准）。
- 若用户只答「是」未给分：按 protocol 仍可更新弧位等；请用户补分后再 **再一次整文件重写**（仍 ≤300 字），补全 Prompt 侧校准。
- 回复中说明 **已重写**、路径与 **约 X 字/300**；**不得**在用户否认或沉默代答时写入。

## 聊天默认交付块

1. **连续性**：2～4 行说明承接与近页差异（**完整选型理由**已写入 md 的 `## 主题与选型理由`）。  
2. **主题（本拍）**一句。  
3. **小红书主标题（3～10 汉字）**。  
4. **即梦正向 prompt（完整可复制）** + **已落盘路径** `作品/{两位序号} {标题}.md`（同文件含 **主题与选型理由** + **即梦 prompt** 存档）。  
5. **（按需）留白对白 / 对白释义 / 发布包装**——遵守 [references/xhs-and-tone.md](references/xhs-and-tone.md)。  
6. **即梦 prompt 0～10 有效性 + 是否并入 `memory.md`（≤300字，合并蒸馏）**——两段询问原文以 [references/memory-protocol.md](references/memory-protocol.md) 为准原样输出。

- [ ] 主题小、可画、与角色不违和
- [ ] 主标题 **3～10 汉字** 且不像 AI 模板腔
- [ ] Prompt 五段顺序与模板一致；`本页情节` **以 `竖版内页。` 起笔**、不重复画风段；钉尾按需且配对正确
- [ ] `作品/{两位序号} {标题}.md` 已写入 **`## 主题与选型理由`**、`## 小红书主标题`、`## 即梦正向 Prompt`（含钉尾若本 run 有）
- [ ] 已询问 **0～10 prompt 有效性** + 是否更新滚动 memory；仅在用户明确肯定后写入
- [ ] 已按 rubric **节 C** 再确认闸门后写入；`memory.md` **原则级合并覆写**、**≤300 字**；**C/B 为主、A 类型化一句**；已做具象扫描；依评分转写 **C 抽象**；未贴全文 prompt、未用剧情/数据/路径顶替原则区
