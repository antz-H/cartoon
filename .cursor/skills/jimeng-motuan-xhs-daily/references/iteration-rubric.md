# 落盘前自检 · 反思闭环（本 skill）

目标：在**无用户参与**的单轮内，用可核对项减少结构错误与同构；必要时**静默修订一轮**再交付。不自评「艺术好坏」，只卡**协议与可执行性**。

---

## A · 即梦正向 Prompt（发给模型前）

| 检查项 | 过线标准 | 不过线时 |
| --- | --- | --- |
| 五段顺序 | 严格：`角色`→`画风`→`留白`（四选一）→`本页情节`（**以 `竖版内页。` 起笔**）→`质量词` | 重排；**勿**把画风段重复塞进 `本页情节` |
| 留白段 | 与 [Prompt-template.md](Prompt-template.md) 所选版本**原文一致**（结构不删改、不合并段） | 对照模板改回 |
| 可画性 | 单场景、主体动作与机位在一句里能说清 | 删抽象形容词，补一个可见动作 |
| 与 memory | 无直接矛盾（机位/否定项/钉尾习惯等） | 以 memory 为准改 prompt，并在「选型理由」里用半句说明取舍 |
| 与近 2～3 页 | 天色 / 机位 / 道具 / 身体关系 **至少变一项**（信息流） | 改 `本页情节` 或选型，并在 `## 主题与选型理由` 写明「相对第 XX 页换了哪一项」 |

**一轮修订规则**：上述任一项不过线 → **只改 prompt 或选型理由文字**，不重起主题；改完 **重新跑 A 表** 一次。仍卡壳（如用户弧与近页冲突）→ 聊天用 1～2 句说明卡点，**禁止**声称已写入 `作品/`。

---

## B · `作品/{两位序号} {标题}.md`（写入后、聊天交付前）

| 检查项 | 过线标准 |
| --- | --- |
| 路径 | 新序号 = 现有最大两位序号 +1；**不得覆盖**已有文件 |
| 三节齐全 | `## 主题与选型理由`、`## 小红书主标题`、`## 即梦正向 Prompt` 顺序正确 |
| 主标题 | 一行，**3～10 汉字**，且不像模板腔（见 [xhs-and-tone.md](xhs-and-tone.md)） |
| Prompt 节 |  fenced 代码块内为**完整**可复制正文；若有钉尾，钉尾含于本节内 |
| 诚实性 | 若落盘失败或未执行，聊天**不得**写「已写入」 |

---

## C · `memory.md`（仅用户明确「是」之后）

执行 [memory-protocol.md](memory-protocol.md) 全文前，额外确认：

1. 已读旧版 `memory.md`（若有）并完成 A/B/C **槽位化**再合并。  
2. 全文件 **≤300 字**（含标点空格）；超出则删并合并同类项。  
3. **禁止**用本轮剧情复述占满 B/C；**禁止**贴整段即梦 prompt。  
4. 用户未确认「是」→ **绝不写** `memory.md`。  
5. **原则优先**：正文主体是 **C/B 可迁移短句**；**无**作品序号、路径、曝光/互动数字作独立信息（除非已抽象成跨话策略一句）。  
6. 已通过 protocol 的 **写回前具象扫描**：删掉「离开本篇就不成立」的句子，或升格为维度级规则。

---

## D · 可选：聊天交付前的「人感」微检（非强制改稿）

- 若附录含对白：去掉「某某说」后仍像两人在接话？否则改两句口语。  
- 若主标题含生造复合词或「雅称」腔：换一个白话短语。

此项不过线可改可不改；**A/B 不过线必须处理**。
