# 叙事 memory 协议

## 定位（必读）

`memory.md` 的用途只有一件：**持续提升**下一轮「定主题 / 写即梦正向 prompt」的质量。它是**跨轮累积的蒸馏指导**，不是「上一拍剧情摘要」，也不是「把用户原话记下来」的剪贴板。

**覆写哲学：原则先于案情**（决定「覆写后长什么样」）

- **应写（优先占字数）**：**可迁移的原则**——下一轮不看本篇 `作品/` 也能用的短句：Prompt **指令形态**（五段里哪类词对你有效/无效、否定项怎么列、钉尾何时用）、**画面维度级**偏好（机位家族、留白策略、雾/阶纹/轮廓等你反复复现的**规则**）、标题的**句式策略**（如首句悬声音不悬评价）。一句话检验：**删掉作品序号后，这句还对下一张有用吗？** 有用才留。
- **少写或极短**：**A 弧位 / 未决**最多 **1 句**，且须 **类型化**（「哪类关系 / 哪条感官线没收口」），**禁止**台账体（篇号+已发+地名拼盘）。连载事实、具体情节以 `作品/*.md` 为准，memory **不复制案情**。
- **默认不写进 memory**（除非已被你压成**跨话**规则且仍值回字数）：作品两位序号、笔记标题全文、文件路径、`xx 观 / xx%` 等**单次运营数据**、单话剧情复述、对白原句。运营教训若要留：**一条抽象策略**（如「封面首句写声响悬念」），**不带**数字报表。
- **要做**：把分数与短反馈**转写**为上述**抽象层**的偏好与避坑；**继承**上一版里仍成立、未被否定的**原则句**，与新认知**合并同类项**后写回。
- **不要做**：用本轮剧情、数据、路径**顶替**旧版里已沉淀的 prompt/画面原则；假装「全新一版」却丢掉可迁移写法。

**「整文件重写」的含义**：因 ≤300 字上限，保存时仍是**覆盖写回整个文件**；语义上必须是**外科手术式合并**（读旧 → 按槽位归并 → 删矛盾与过期 → 再写回），**禁止**空档重启、禁止默认「本轮优先所以清空旧指导」。

**篇幅硬约束**：整个 `memory.md` 文件，从文件开头到末尾，**总字数 ≤300 字**（汉字、字母、数字、标点、空格均计入；与常见编辑器「字符数 / 字数」统计一致）。超出则必须**删旧留新、合并同类项**；删的时候优先删：**单次运营数据与路径、仅当话有意义的具象台账**、**已随 `作品/` 推进而过期的弧位一句**、**被用户明确推翻的旧偏好**、**与更高分轮次冲突的弱句**——**不要**优先删「可迁移的 prompt/画面原则」。

**禁止**：贴整段即梦 prompt、抄长对白、按篇罗列剧情全文。细节与完整 prompt **只在** `作品/{序号} ….md`。

## 读

每次开跑先读（若存在）：

- `memory.md`（仓库根目录，与 `作品/` 同级，即本仓库 `cartoon/memory.md`；若用户改用其他路径，以用户指定为准）

同时读连续性来源（**默认**以下项；能代表正序者优先）：

- 近 2～3 篇 `作品/*.md` 中的「本页情节」与留白

若用户在本轮消息中给出弧/主线大纲（路径或粘贴正文），**追加**阅读，不作为仓库内默认必备文件。

若 `memory.md` 尚不存在：开跑阶段**不创建**；仅在用户确认「是」后，按「写」**新建并写入**（仍须 ≤300 字）。不要跳过「询问 **0～10 分** + **是否写入**」步骤。

若用户在本轮消息中粘贴**前一日（或近期）外露数据**（曝光、封面点击率、互动、停留、搜索词、负评摘录等）：视为**可选追加语境**，与 `memory.md`、近页 `作品/` **一起读**；**不得**在未蒸馏、未获用户确认写入前，把原始数字表整段抄进 `memory.md`。

## 前一日数据反馈（怎么用）

**你怎么给**：每日开跑时，在**同一条或紧随其后的消息**里粘贴即可（截图转文字、后台导出、自记表格均可）。本 skill **不要求**单独落盘文件；你若要留档可自行维护表格或笔记，仓库不强制路径。

**Agent 怎么用（当天）**

1. **对照**：把数据与**对应那一篇** `作品/*.md` 里的主标题、留白首句/钩子类型、即梦 prompt 里与**封面/缩略图**相关的机位与外亮条等**对上号**，区分是「标题不点」「首图缩略糊/同构」「正文跳出」还是样本太小。  
2. **弱信号原则**：单日波动大，**不**用一条数据推翻连载弧与可画性主目标；只用来**微调维度**（标题句式、首句是声响还是评语、顶标区是否够亮、是否需更强外亮锚点等）。  
3. **与即梦 prompt 分的关系**：**0～10** 仍是「控图/出图」口径；**笔记数据**是「分发侧」口径。二者矛盾时，在聊天里用一两句说明取舍，**memory 里分别抽象成两条短原则**（不占剧情字数）。

**写入 `memory.md` 时（仅用户确认「是」之后）**

- 遵守上文 **覆写哲学**：把数据**蒸馏成一条策略或避坑**，**不写**曝光/百分比/具体路径原文（例：好 — 「低封点时首句用声响悬念，少用评语体起笔」；不好 — 「562 观 13 封点 2%」整行搬进 memory）。  
- 若你**明确要求**保留某个对比口径：仍尽量压成 **≤1 句** 且带**相对词**（如「仍偏低」），并准备下一轮用新数据**替换**该句，避免 memory 变成报表栏。  
- **具象扫描**一步：若写回稿里出现纯数字串、文件名，一律删或改写成原则句。

## 询问（必须）

在聊天中输出本次 **主题一句、主标题、即梦 prompt 要点、可选小红书附录** 后，**依次**询问用户（两段都须出现；可放在连续两行）：

1. **即梦 prompt 有效性（0～10）**——衡量**发给即梦的整段正向 prompt** 对你控图、出图的帮助程度（不是评价小红书正文）。原文建议：

> 请给本次 **即梦正向 prompt 有效性** 打分：**0～10**（0=几乎无效或严重跑偏，10=高度贴合、你只需微调即可出片）。分数会用来**调整** `memory.md` 里关于「怎么写 prompt 才有效」那几句总结。

2. **是否更新滚动 memory**——原文建议：

> 是否把本次反馈**并入** `memory.md` 的**滚动总结**（整文件总字数 **≤300 字**；保存时为覆盖写回，但会与旧版**合并蒸馏**为**原则级、可迁移**短句，用于改进之后定主题与写 prompt；**不写**篇号数据路径、不单话复述剧情）？回复 **是 / 否**。

用户可一条消息合并回复，例如：`8 分，是` / `是，7`。

**不得**在用户明确说「是」之前写入或改写该文件。

### 若用户只答了「是 / 否」而未给分

- 仍可执行一次「写」：把本轮**类型化未决 / 抽象走向**并入 **A**（禁台账）；**B/C 仍以跨话原则为主**，**Prompt 侧少写或不写推断**。
- 在聊天中请用户补 **0～10 分**；补分后若用户再次确认可写，再**整文件重写**同一份总结（仍 ≤300 字），把评分反映进去。

## 写 · 用户确认「是」之后（整文件重写）

### Agent 合并写入前自检（≤10 秒）

- 用户本轮是否**明确**肯定写入（「是」/「8 分，是」等）？**否** → 停，不写。  
- 是否已读旧版全文并完成 **A/B/C 槽位**归类（见下节步骤 2）？**否** → 先读后写。  
- 预估写回后是否 **>300 字**？**是** → 先删并合并同类项再落笔。  
- 是否计划把 **整段即梦 prompt、本轮剧情复述、篇号/路径/运营数据**当正文主体？**是** → 禁止；改为**原则级**可迁移偏好/避坑。

1. **读取**当前 `memory.md`（若存在）与本 run 的：主标题、主题一句、未决钩子、用户 **0～10 分**（若有）、用户额外一句反馈（若有）。
2. **槽位化旧文（脑中即可，不必在文件里写小标题）**：把旧版句子粗分为三类，避免合并时「整段当剧情扔掉」：
   - **A 弧位 / 未决钩**：**≤1 句**；只保留**类型化**未决（哪类张力/哪条感官线没收口）；**避免**篇号、发帖记录、地名坐标式台账。随连载推进**允许**替换；旧钩已落地则删。
   - **B 选题 / 画面偏好**：**维度级**规则（机位家族、留白策略、雾/轮廓/材质等你跨话复用的**原则**），**禁止**单话道具白描占篇幅；**默认继承**，除非用户本轮明确否定或与近 2～3 篇 `作品/` 矛盾。
   - **C Prompt 写法**：**指令形态级**（五段习惯、否定项列法、钉尾、词密度）；本轮分数与一句反馈只**抽象**成一条强化/避坑，**禁止**把半段情节当「prompt 技巧」写进来；**默认继承**，不整带清空。
3. **合并蒸馏**：旧 A/B/C + 本轮增量 → 新正文；**同一意思只留一句**；**禁止**用剧情、数据、路径占满 B、C。
3b. **写回前具象扫描（强制）**：定稿前过一眼——是否含**作品序号、路径、曝光/互动数字、仅本篇成立的道具句**？有则**删或升格为跨话原则**（升格后仍须通过「删掉序号还对吗」检验）。
4. **一次性写回**整个 `memory.md`（覆盖旧版），**全文件 ≤300 字**。
5. 回复中说明 **已重写**、路径，并注明 **当前约 X 字 / 上限 300**（X 为 agent 自检计数）；**建议**用半句说明本轮是「延续旧习惯 / 修正了哪一类一句」（不写长解释）。

### 建议内容块（非模板填空，可删可并，服从 300 字）

**字数分配优先级**（从多到少）：**C Prompt 原则** > **B 画面维度原则** > **标题句式原则** > **A 类型化未决一句**。不要把 300 字浪费在**对下一帧 prompt 无帮助**的案情与数据上。

- **A 弧位 / 未决**：**一句、类型化**；不写台账。
- **B 画面与叙事**：跨话可用的**规则**（机位/留白/雾阶纹轮廓等你个人反复踩的**类**），禁单话道具清单。
- **C Prompt 有效习惯**：高分沉淀的**半句到两句指令级**写法；低分**一条抽象避坑**（不要失效 prompt 原文）。
- **标题 / 人感**（若常反馈）：各 **≤1 条策略级短句**（非某篇标题照抄）。

### 依评分调整「Prompt 指导」侧（仍计入 300 字）

在合并时参考本轮分数（**衡量 prompt 有效性**）：

| 分数 | 合并倾向 |
| --- | --- |
| **9～10** | 把本轮**最管用的一条写法**压成**脱离本篇仍成立**的短语写进 **C**（如钉尾/机位/否定项）；删旧条目中较弱的同主题句以腾字数。 |
| **7～8** | **轻量强化**：用几个词到半句肯定「这类指令对你有效」。 |
| **4～6** | **不写成本轮成功学**；若有用户说明，压缩成 **1 条待改进**（如「俯拍易糊阶纹」）。 |
| **0～3** | **避坑优先**：用 **1 短句**写「少用什么 / 别重复什么」，避免把失败 prompt 原文搬进 memory。 |

若本轮 **未提供分数**：总结里 **不写**具体分数；Prompt 侧不写基于分的推断，可留「待补分再校准 prompt 句」类 **≤10 字** 占位，或干脆不占——由 agent 在 300 字内取舍。

## 写 · 首次（`memory.md` 尚不存在且用户已确认写入）

首行建议：`# 墨团与芽仔 · memory（滚动指导）`（计入 300 字）。其下直接写**原则级蒸馏正文**（**C/B 为主**、**A 一句类型化**），**整文件仍 ≤300 字**。**不要**写「由 agent 追加」类暗示流水账的副标题，除非该句本身承担指导意义且未超字数。
